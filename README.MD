
# Web Components Chat Filter Scanner

## Описание проекта

Этот проект представляет собой исследование возможностей **веб-компонентов (Web Components)** - набора нативных технологий браузера для создания переиспользуемых кастомных HTML-элементов. Основная цель - реализовать компонентный подход, аналогичный React, используя только ванильный JavaScript без дополнительных фреймворков.

## Цели исследования

1. **Изучение Web Components API**:
   - Custom Elements API для создания кастомных HTML-тегов
   - Shadow DOM для изоляции стилей и структуры компонентов
   - События и коммуникация между компонентами

2. **Реализация реактивного взаимодействия**:
   - Создание компонентов с собственным состоянием
   - Реализация двустороннего связывания данных
   - Организация коммуникации между компонентами через события

3. **Создание практического приложения**:
   - Веб-интерфейс для взаимодействия с backend-сервером
   - Инструмент для фильтрации и обработки HTML-контента

## Функциональность приложения

Приложение представляет собой **сканер сохраненных HTML-страниц** из чата ChatGPT с возможностью их фильтрации и сортировки. Основные возможности:

### Основные функции:
- **Сканирование HTML-файлов**: Анализ сохраненных веб-страниц
- **Фильтрация контента**: Удаление нерелевантных тем и разделов
- **Сортировка материала**: Организация информации по категориям
- **Формирование документации**: Создание структурированной и понятной базы знаний

### Решаемые проблемы:
- **Очистка контента**: Исключение мусорной информации из сохраненных страниц
- **Структурирование**: Преобразование неорганизованного HTML в полезную документацию
- **Категоризация**: Автоматическая сортировка материала по темам

## Технологический стек

### Основные технологии:
- **Vanilla JavaScript** (ES6+)
- **Web Components API**:
  - Custom Elements
  - Shadow DOM
  - HTML Templates
  - Custom Events
- **CSS3** с изоляцией стилей через Shadow DOM

### Архитектурные подходы:
- Компонентно-ориентированная архитектура
- Event-driven дизайн
- Изолированные модули с четкими интерфейсами

## Структура компонентов

Проект реализует следующие кастомные элементы:

### `checkbox-list`
Компонент для отображения списка с множественным выбором:
- Динамическая отрисовка чекбоксов из переданных данных
- Отслеживание выбранных элементов
- Генерация событий при изменении состояния
- Методы для получения выбранных индексов и значений

## Преимущества подхода

### По сравнению с фреймворками:
1. **Нет зависимостей** - работает в любом современном браузере
2. **Меньший размер бандла** - отсутствие runtime библиотек
3. **Нативная производительность** - прямое использование API браузера
4. **Долговечность** - не зависит от жизненного цикла фреймворков

### Для обучения:
- Глубокое понимание основ веб-платформы
- Прямая работа с DOM API
- Понимание принципов изоляции и инкапсуляции компонентов

## Назначение проекта

Этот проект служит как:
1. **Исследовательская площадка** для изучения возможностей веб-компонентов
2. **Практический инструмент** для обработки сохраненных материалов из ChatGPT
3. **Демонстрация** того, что сложные интерфейсы можно создавать без фреймворков

Результатом работы является не только функциональное приложение, но и глубокое понимание того, как работают современные веб-технологии на фундаментальном уровне.
